💡 Essential DAX Measures to Create

Total Patients = COUNTROWS(Patients) | Total Revenue = SUM(Hospital Bills[total_amount]) | Avg Length of Stay = AVERAGEX(Patients, DATEDIFF([admission_date],[discharge_date],DAY)) | Bed Occupancy % = DIVIDE(COUNTROWS(FILTER(Beds,[status]="Occupied")), COUNTROWS(Beds)) * 100 | Avg Satisfaction = AVERAGE(Satisfaction Score[rating])

⚡ Performance Optimisation Requirements

✅ Do These
  • Use measures instead of calculated columns for aggregations
  • Remove unused columns before loading into data model
  • Use star schema (avoid snowflake for simplicity)
  • Create a separate Date dimension table
  • Disable auto-date/time in Power BI settings
❌ Avoid These
  • Avoid bi-directional relationships (use single direction)
  • Do not import all columns — only needed ones
  • Avoid many-to-many relationships where possible
  • Do not use CALCULATE inside SUMX unnecessarily

Interview Tips & Talking Points

How to present and explain this project confidently to interviewers and students

🎤 The Perfect 2-Minute Project Summary

"I built a multi-page Hospital Data Analysis dashboard in Power BI by integrating 15 different datasets — covering patients, billing, doctors, beds, inventory, and satisfaction. I performed data cleaning using Power Query, built a Star Schema data model with Fact and Dimension tables, created 30+ DAX measures, and designed 8 interactive dashboard pages. The key insight I found was that ICU beds were at 90% occupancy while paediatrics was at 55% — indicating a resource rebalancing opportunity."

❓ Q: Why did you choose Star Schema?

✅ "Star Schema provides better query performance in Power BI. The Fact tables contain transaction data with foreign keys, and Dimension tables contain master data. It avoids redundancy, enables faster aggregations, and makes DAX calculations more efficient."

❓ Q: What challenges did you face in data cleaning?

✅ "The main challenge was handling NULL values in discharge_date for active patients — I couldn't replace them with 0, so I kept them as-is and used ISBLANK() in DAX to filter. Also standardising inconsistent gender values (M/Male/male) required custom column logic."

❓ Q: What is your most complex DAX measure?

✅ "Bed Occupancy Rate: DIVIDE(COUNTROWS(FILTER(Beds, [status]="Occupied")), COUNTROWS(Beds)) * 100. I also built a 'Running Total Revenue' using TOTALYTD and a 'MoM Revenue Change %' using DATEADD."

❓ Q: What business insight did you find?

✅ "Three key insights: 1) ICU had 90% bed occupancy — suggesting need for expansion. 2) 23 medicines below reorder level — inventory risk. 3) Emergency dept had lowest satisfaction score (3.2/5) — flag for quality improvement."

❓ Q: Why do you use measures vs. calculated columns?

✅ "Measures are evaluated at query time using filter context — they're more flexible and don't consume memory. Calculated columns are computed at refresh time and stored in the model. For aggregations like SUM/AVG/COUNTROWS, measures are always preferred."

❓ Q: How would you improve this project further?

✅ "I would add: 1) Row Level Security so department heads see only their data. 2) Predictive analytics for patient readmission risk. 3) Real-time data refresh using Power BI gateway. 4) Mobile-optimised dashboard layout."

🏆 Tips for Presenting to Students / Beginners

Start with the business problem (why this exists) → show the raw data (Excel files, messy) → show Power Query transformations step by step → explain the data model diagram → walk through 2–3 dashboard pages highlighting insights. Always connect each visual back to a business decision it helps make.
